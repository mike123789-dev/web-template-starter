# Glossary

- `PRD`: 제품 목표/요구사항을 정의한 source of truth 문서
- `Spec`: 기능 동작, 범위, acceptance criteria를 정의한 문서
- `Plan`: 기술 선택, 구현 전략, 리스크 문서
- `Tasks`: 실행 단위 작업과 테스트 명령 문서
- `Task Detail`: 특정 task의 상세 범위/검증/의존성을 담은 문서 (`tasks/T-xxx.md`)
- `Test Matrix`: 요구사항/AC와 테스트 간 매핑 문서
- `Feature ID`: 기능 식별자(`F-001` 형식)
- `Requirement ID`: PRD 요구사항 식별자(`FR-001`, `NFR-001`)
- `AC`: Acceptance Criteria
- `DoD`: Definition of Done, 완료 기준 체크리스트
- `Gate`: 반드시 통과해야 하는 검증 명령(예: `npm run verify`)
