- [[../tasks]]
- [[../spec]]

## Objective

프로젝트 목록 영역에서 로딩/오류/빈결과 상태를 사용자에게 명확히 전달하고, 빈결과 시 필터 초기화 액션까지 제공한다.

## Scope

### In

- `ProjectsList` 상태 분기(성공/오류/빈결과) 구현
- 필터 적용 빈결과 메시지 및 `Reset filters` 액션 추가
- `ProjectsListSkeleton` 접근성 속성(`aria-busy`, 안내 텍스트) 반영

### Out

- 백엔드 재시도 정책
- 새로운 API 엔드포인트 추가

## Dependencies

- T-001에서 확정된 AC 문구 및 UI 정책

## Implementation Notes

- 필터 정규화 결과를 빈결과 메시지와 CTA 노출 조건에 재사용한다.
- 오류 상태는 페이지 전체가 아니라 목록 컨테이너 안에서 복구 가능하게 처리한다.
